## ЗАДАЧА

### 1. Найди причину критического бага
Симптом: Consilium получает 200 OK от провайдера, формирует JSONResponse с choices[0].message.content, возвращает Hermes. НО Hermes пишет "Provider failed after retries".

### 3. Ключевые изменения в Hermes v0.19 (по сравнению с v0.18)

**Delivery-obligation ledger:** Ответы сохраняются в `state.db`. Если gateway упал между генерацией и доставкой — ответ НЕ теряется, повторно доставляется при следующем запуске.

**Tool definitions fix:** Исправлена ошибка из-за которой tool definitions не передавались custom провайдерам. В v0.19 tools пробрасываются корректно.

**Custom provider validation:** Эндпоинт `/v1/models` должен возвращать валидный JSON. Если возвращает ошибку — Hermes может отклонить запрос.

**Provider timeouts:** Новая иерархия таймаутов: `providers.<id>.request_timeout_seconds` → `providers.<id>.models.<model>.timeout_seconds`.

**Prompt caching:** Автоматическое `cache_control` с TTL 1h для Claude/OpenRouter, 5m для Qwen Cloud.

**Tool-loop guardrails:** Детекция застрявших циклов tool calls с hard stops.

**Gateway hot-reload:** Изменения `model.context_length` и `compression.*` применяются без рестарта.

**Compression config migration:** `compression.summary_*` → `auxiliary.compression.*`.

**Per-task fallback chain:** Каждая auxiliary задача может иметь свой `fallback_chain`.

**Обязательные поля ответа (OpenAI Chat Completions API):**
- `id`: строка
- `object`: "chat.completion"
- `created`: timestamp
- `model`: название модели
- `choices[0].message.role`: "assistant"
- `choices[0].message.content`: строка или null (если есть tool_calls)
- `choices[0].message.tool_calls[].id`: UUID
- `choices[0].message.tool_calls[].type`: "function"
- `choices[0].message.tool_calls[].function.name`: имя функции
- `choices[0].message.tool_calls[].function.arguments`: JSON-строка
- `usage`: опционально

### 4. Проверь совместимость Consilium с v0.19
- Формат JSON ответа — соответствует ли всем обязательным полям?
- `/v1/models` — возвращает ли валидный JSON?
- `tool_calls[].id` — UUID или наш формат `call_rescued_{index}`?
- `content` — null или пустая строка при tool_calls?
- `finish_reason` — "tool_calls" при наличии tool_calls?
