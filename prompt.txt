## ДАННЫЕ ПО HERMES AGENT v0.19 ("The Quicksilver Release")

### Ключевые изменения (по сравнению с v0.18):

**Delivery-obligation ledger:** Ответы сохраняются в `state.db`. Если gateway упал между генерацией и доставкой — ответ не теряется, повторно доставляется при следующем запуске. Это может влиять на то как Hermes проверяет успешность запроса.

**Tool definitions fix:** Исправлена ошибка из-за которой tool definitions не передавались custom провайдерам. В v0.19 tools пробрасываются корректно.

**Custom provider validation:** Эндпоинт `/v1/models` должен возвращать валидный JSON с `context_length`. Если возвращает ошибку — Hermes может отклонить запрос.

**Provider timeouts:** Новая иерархия таймаутов: `providers.<id>.request_timeout_seconds` → `providers.<id>.models.<model>.timeout_seconds`. Старый `HERMES_API_TIMEOUT` deprecated.

**Prompt caching:** Автоматическое `cache_control` с TTL 1h для Claude (native Anthropic, OpenRouter, Nous Portal), 5m для Qwen Cloud.

**Tool-loop guardrails:** Детекция застрявших циклов tool calls с hard stops.

**Gateway hot-reload:** Изменения `model.context_length` и `compression.*` применяются на следующем сообщении без рестарта.

**Compression config migration:** `compression.summary_*` → `auxiliary.compression.*`.

**Per-task fallback chain:** Каждая auxiliary задача может иметь свой `fallback_chain` с provider/model/base_url.

**Аудит показал:** Ответ Consilium содержит лишние поля (`provider`, `system_fingerprint`, `service_tier`) которые Hermes v0.19 может отбраковывать. `tool_calls[].id` должен быть UUID а не `call_rescued_N`. `/v1/models` должен возвращать `context_length`.

### Обязательные поля ответа (OpenAI Chat Completions API):
- `id`: строка
- `object`: "chat.completion"
- `created`: timestamp
- `model`: название модели
- `choices[0].message.role`: "assistant"
- `choices[0].message.content`: строка или null (если есть tool_calls)
- `choices[0].message.tool_calls[].id`: UUID
- `choices[0].message.tool_calls[].type`: "function"
- `usage`: опционально
