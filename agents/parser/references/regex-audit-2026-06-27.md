# Regex Audit — 26.06.2027

## Критические баги

### 1. `Е` vs `Ё` в character class — Ё НЕ перехватывается

**Файл:** `pipeline_v55_final.py`, `cleaner/scorer.py`

**Баг (был):**
```python
BAD_NO_PRODUCT = re.compile(
    r'напитки|безалкогольн...|[^а-я]энергетическ|[а-я]{1,3} абсурд',
    re.I
)
```

**Проблема:** `[^а-я]` — это отрицание БЕЗ `ё`! Регистр `re.I` делает паттерн нечувствительным к регистру, но НЕ добавляет `ё`/`Ё` в набор. Результат:
- Компании типа `ООО Промышленные напитки` на `ООО ПромышленнЫЕ напитки` — НЕ отсекались если `ы` стоит после `е`
- `[^а-я]энерг` — найдёт `1энерг` или `,энерг`, но НЕ найдёt `ёэнерг` (хотя `ё` стоит в начале слова, а не перед `энерг` — это не та проблема)
- И главное: слово `промышленн` с буквой `ё` (`промышленный`) — `ре.I` не делает `е` ≈ `ё`. Это ДВА РАЗНЫХ символа.

**Исправлено:**
```python
BAD_NO_PRODUCT = re.compile(
    r'напитки|безалкогольн...|[^а-яёЁ]энергетическ...',
    re.I
)
```

**Правило:** **ВСЕГДА** включай `ё` и `Ё` в character class `[^...]` при `re.I`. Или используй `а-яА-ЯёЁ` полностью.

### 2. Жадные regex без lookahead

**Файл:** `pipeline_v55_final.py`

**Баг (был):**
```python
RE_ADDRESS = re.compile(r'\s*(?:офис|кв\.|комн\.)\s+\d+.*')
RE_NOT_COMPANY = re.compile(r'\b(?:выставк|ярмарк|фестивал|конференц|конгресс|саммит|форум)\w*\b', re.I)
```

**Проблема:** `RE_ADDRESS` с `.*` захватывает ВСЁ до конца строки. Если в названии есть "офис 123, менеджер Иван" — удалится всё после "офис 123".

**Исправлено:**
```python
RE_ADDRESS = re.compile(r'\s*(?:офис|кв\.|комн\.)\s+\d+(?=\s|$|[^\w\s])')
```

**Правило:** Используй lookahead `(?=...)` вместо жадного `+`/`*` когда нужен точный захват.

### 3. Избыточные проверки

**Было:**
```python
if email and email != "":
```

**Стало:**
```python
if email:
```

**Правило:** `if x and x != ""` → `if x` (пустая строка = falsy).

## Паттерны мусора — что должно быть в BAD_KEYWORDS

### НЕ удалять (оставить в BAD):
- `СТЕНД`, `STAND`, `BOOTH` — "СТЕНД 82B88" это мусор выставки
- `PAV`, `HALL`, `PAVILION` — то же самое

### Добавить в BAD/NON_FOOD:
- `РЕСПУБЛИКА`, `ОБЛАСТЬ`, `КРАЙ`, `ГОРОД`, `АДРЕС` — географические термины в начале названия
- `EXPO`, `FAIR`, `CONFERENCE` — на английском
- `ВЫСТАВКА`, `ВЫСТАВОЧНЫЙ` — выставочные термины

### Примеры мусора который должен отсекаться:
- "СТЕНД 82B88" → score 35 → grey zone → fuzzy dedup
- "33-я МЕЖДУНАРОДНАЯ ВЫСТАВКА" → score 0 → low_score
- "РЕСПУБЛИКА УЗБЕКИСТАН" → score 5 → low_score
- "АЛТАЙСКИЕ ГРЕНКИ" → score 15 → grey zone (под вопросом)
