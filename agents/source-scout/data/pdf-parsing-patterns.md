# PDF Parsing Patterns (06.2026)

## Библиотека
**PyMuPDF (fitz)** — быстрее PyPDF2, поддерживает `get_text("words")` с координатами.

## Паттерн 1: Таблица с колонками по x-координатам
**Когда**: PDF с таблицей, `get_text()` возвращает неструктурированный текст.
Использовать `get_text("words")` → группировка по (block_no, line_no) → фильтрация по x.

## Паттерн 2: Запись = несколько строк в одном block
**Когда**: Каждая запись занимает 2-3 строки.
Собирать line_data с типами ('country', 'name', 'stand', 'page'), обрабатывать последовательности.

## Паттерн 3: PDF Viewer (pdf.js)
**Когда**: Сайт использует PDF viewer — ссылка ведёт на HTML, а не на PDF.
Playwright → intercept responses → найти реальный URL в `mediafile/`.

## Паттерн 4: DataTables (jQuery)
**Когда**: Сайт использует jQuery DataTables с AJAX-пагинацией.
`page.evaluate()` для извлечения из DOM + изменение `select[name*='length']` на 100.

## Частые ошибки
1. PyPDF2 таймаутится на больших PDF → использовать PyMuPDF
2. Playwright EPIPE при >25 страниц → ограничить max_pages=10
3. DataTables пагинация не кликается → evaluate + select[name*='length']
4. Всё капсом в PDF → страны отличаются по наличию стенда на след. строке
5. Дедупликация между годами → НЕ удалять дубликаты по имени между годами
