## torgbox.ru — парсинг

- **~500 записей** в temp_clients имеют `website LIKE '%torgbox.ru%'`
- Карточки контрагентов: ИНН, ОГРН, адрес, ОКВЭД — всё структурировано
- **ИНН парсится** из `panel-content-list__label` / `panel-content-list__value`
- **Телефон НЕ доступен** — только 8-800 номер площадки (требует авторизации)

## Serper — оптимизация (11.06.2026)

**Проблема**: при массовом поиске credits быстро заканчиваются (400 "Not enough credits") или ключ невалидный (403 "Unauthorized").

**Оптимизация**:
1. **Короткие запросы**: `f"{name} {country} телефон"` — НЕ `f"{name} {country} телефон email контакты сайт"`
2. **Фильтрация ДО поиска**: исключить нецелевые компании (журналы, оборудование, мясо, рыба, услуги, логистика, консалтинг, агентства и т.д.)
3. **Фильтрация мусорных доменов**: не искать для записей с website вроде workius.ru, maptons.com, cibum.ru — это не реальные сайты компаний
4. **Проверка ключа перед запуском**: тестовый запрос `{"q": "test", "num": 1}` — если 403 или 400, не запускать массовый поиск
5. **Сохранять прогресс**: писать промежуточные результаты в файл каждые 50 записей

**Статистика Serper (10-11.06.2026)**:

- Вторая волна: 0 обновлено (403 Unauthorized — невалидный ключ)
- Вывод: проверять ВСЕГДА перед запуском

## Supabase — таймауты и retry (11.06.2026)

- Supabase может таймаутить при загрузке >1000 записей — использовать offset/limit пагинацию с timeout=60 и retry
- Использовать `requests.Session()` с `Retry(total=3, backoff_factor=2)` для надёжности
- Не запускать длительные скрипты в `execute_code` — они прерываются таймаутом. Использовать `terminal(background=true)` с `nohup`

## Рекомендуемый pipeline обогащения

1. `parse_torgbox.py` → ИНН для всех torgbox-записей
2. `scrape_phones.py` → телефоны с реальных сайтов (~46% успех)
3. DaData findById (платный) → телефоны по оставшимся ИНН
