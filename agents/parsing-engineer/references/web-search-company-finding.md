# Web Search Company Finding Method — 21.06.2026

## Контекст
Прямой парсинг сайтов (curl/requests) часто не работает из-за Cloudflare, SPA, SSL-ошибок. Для стран с малым количеством записей (TJ, TM, KG, UZ, AZ, AM, GE) наиболее эффективен **web_search + нормализация**.

## Рабочий метод

### 1. Поиск компаний
Запросы для web_search по странам:
- Таджикистан: "таджикистан кондитерская фабрика завод производитель", "таджикистан сухофрукты экспорт компания", "Исфара сухофрукты производитель"
- Узбекистан: "узбекистан кондитерская фабрика завод", "узбекистан мясокомбинат молочный завод"
- Азербайджан: "азербайджан кондитерская фабрика завод производитель"
- Армения: "армения пищевые производители экспорт компания"
- Грузия: "грузия пищевые производители каталог"
- Кыргызстан: "кыргызстан кондитерская фабрика завод"
- Туркменистан: "туркменистан пищевые производители экспорт"

### 2. Извлечение названий
- Искать конкретные названия в description результатов
- Фильтровать: должны содержать ООО/АО/завод/фабрика/производитель
- Исключать: российские компании, не пищевые

### 3. Нормализация и загрузка
Все записи должны иметь ОДИНАКОВЫЙ набор ключей (PGRST102!):
FIELDS = ['name', 'country', 'city', 'address', 'phone', 'email', 'website', 'description', 'categories', 'source', 'source_year', 'raw_data']
Каждая запись = {f: "" for f in FIELDS} + заполненные значения

## Питфоллы
- PGRST102: одинаковый набор ключей обязателен
- Русские символы в URL: использовать urllib.parse.urlencode()
- Cloudflare: многие сайты защищены, не парсить напрямую
- Непищевые компании: каталоги типа GeorgiaYP возвращают IT/логистику
