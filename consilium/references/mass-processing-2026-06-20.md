# Consilium — Массовая обработка (20.06.2026)

## Паттерн: needs_review.json → Консилиум → clean_clients

### Скрипты
- `layer4-cleaner/scripts/process_review.py` — обработка батчами по 20, лимит 1000 за запуск
- `layer4-cleaner/scripts/process_all_review.py` — обработка всех записей за один проход (без лимита)

### Алгоритм
1. Загрузить `needs_review.json` (записи с неопределённым статусом)
2. Загрузить существующие `(name_clean, country)` из `clean_clients`
3. Отфильтровать дубли (уже есть в базе)
4. Отправить батчи по 20 в консилиум (3 модели)
5. Консилиум решает: is_company (да/нет), страна, категория, confidence
6. Записи с confidence > 0.5 и целевой страной → в `clean_clients`
7. Кэшировать решения в `data/consilium_cache.json`
8. Записи где консилиум не уверен → отбрасываются

### Производительность
- ~1 батч (20 записей) за 10-15 секунд
- 500 записей ≈ 40-60 минут (при 2 работающих моделях)
- 1000 записей ≈ 1.5-2 часа

### Circuit Breaker — КРИТИЧЕСКАЯ ПРОБЛЕМА
**Проблема**: При массовой обработке (100+ запросов) Groq и Gemini отключаются circuit breaker'ом после 5 ошибок подряд и не успевают восстановиться.

**Текущие настройки** (обновлено 20.06.2026):
- Порог: 10 ошибок подряд (было 5)
- Блокировка: 120 секунд (было 300)

**Рекомендации**:
- Для массовой обработки (10K+ записей): увеличить порог до 20 ошибок или отключить circuit breaker
- Использовать только 2 стабильные модели (Mistral + Gemini) чтобы минимизировать ошибки
- Таймаут на модель: минимум 90s для Mistral
- Добавлять `await asyncio.sleep(1)` между батчами чтобы не перегружать API

### Кэширование
Решения консилиума кэшируются в `data/consilium_cache.json` по ключу `{name_clean}|{country}`. При повторном запуске — мгновенная обработка без траты токенов.

### Фильтрация существующих
Перед загрузкой в `clean_clients` — всегда загружать существующие ключи и фильтровать дубли ДО insert. Функция `load_existing_clean_clients()` в скриптах слоя 4.

### unique constraint в clean_clients
`clean_clients_name_country_uniq` на `(name_clean, country)` — INSERT падает с ошибкой 23505.
**Решение**: Загружать существующие ключи и фильтровать ДО insert.

### TRUNCATE через RPC
```python
sb.rpc("exec_sql", {"sql_query": "TRUNCATE clean_clients"})
```
DELETE по одному через `.eq()` слишком медленно для 10K+ записей.

### _extract_json — обработка массивов
Модели часто возвращают JSON-массив `[{...}, {...}]`, а не объект `{...}`.
`_extract_json()` теперь сначала ищет `[...]`, потом `{...}`.
