# Технические заметки — 20.06.2026

## Критические баги, исправленные в этой сессии

### 1. consilium_ask() отсутствовала в providers.py
**Симптом:** `from providers import consilium_ask` — ImportError, все агенты (enricher, parser, matcher, consilium_brain) падали.
**Причина:** `__init__.py` импортировал `consilium_ask` из `providers`, но функция не была определена.
**Решение:** Добавлена полная реализация в `providers.py` — параллельный опрос моделей, консенсус, robust JSON extraction.

### 2. _extract_json() не парсила JSON-массивы
**Симптом:** Консилиум возвращал `[{"idx": 1, ...}]`, но `json_result` был `None`.
**Причина:** `_extract_json` искала только `{...}`, не `[...]`.
**Решение:** Добавлен поиск массивов перед объектами.

### 3. _ask_cloudflare() была сломана
**Симптом:** Cloudflare всегда возвращал None.
**Причина:** Тело функции (session.post) было потеряно при предыдущем редактировании.
**Решение:** Восстановлена.

### 4. clean_clients unique constraint
**Симптом:** INSERT падал с ошибкой 23505 (duplicate key).
**Причина:** Unique constraint `clean_clients_name_country_uniq` на `(name_clean, country)`.
**Решение:** `load_existing_clean_clients()` — загрузка существующих ключей перед загрузкой.
