ВОПРОС: Должны ли мы подстраивать Consilium под каждую версию Hermes?

ОТВЕТ: НЕТ. Решение — версионно-независимая система через строгое следование 
       OpenAI API Specification.

КАК В FREELLMAPI РЕШЕНА ЭТА ПРОБЛЕМА:

  FreeLLMAPI использует подход "Strict API Compliance":
  
  1. Возвращает ТОЛЬКО поля из официальной спецификации OpenAI:
     - id, object, created, model, choices, usage
     - choices[0].message: role, content, tool_calls
     - tool_calls[].id (UUID), type, function.name, function.arguments
  
  2. НЕ возвращает проприетарные поля:
     - provider, system_fingerprint, service_tier — удаляются фильтром
  
  3. Динамическая адаптация через заголовки:
     - X-API-Version: версия API клиента
     - X-Feature-Flags: включение/отключение фич

РЕКОМЕНДАЦИЯ ДЛЯ CONSILIUM:

  Добавить заголовок совместимости и version detection:
  
    @app.post("/v1/chat/completions")
    async def chat_completions(...):
        # Detect Hermes version from User-Agent or header
        hermes_version = request.headers.get("X-Hermes-Version", "unknown")
        logger.info(f"Hermes version: {hermes_version}")
        
        # Return ONLY OpenAI standard fields
        response = {
            "id": f"chatcmpl-{uuid.uuid4()}",
            "object": "chat.completion",
            "created": int(time.time()),
            "model": target_model,
            "choices": [{
                "index": 0,
                "message": {
                    "role": "assistant",
                    "content": normalized_content,
                    "tool_calls": tool_calls or []  # Всегда список
                },
                "finish_reason": finish_reason
            }],
            "usage": usage  # Опционально
        }
        
        # Фильтр: только стандартные поля OpenAI API
        std_fields = {"id", "object", "created", "model", "choices", "usage"}
        response = {k: v for k, v in response.items() if k in std_fields}
        
        return JSONResponse(response)
